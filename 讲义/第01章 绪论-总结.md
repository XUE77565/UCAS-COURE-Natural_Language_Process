## 一、基本概念

### 1. 问题提出

- **什么是语言？**
  - 《现代汉语词典》：语言是人类特有的、用来表达意思、交流思想的工具，是一种特殊的社会现象，由**语音、词汇、语法**构成一定的系统。
  - 吴蔚天（1994）：语言可以被看成一个**抽象的数学系统**。
- **什么是自然语言？**（区别于人工语言 / 形式语言 / 编程语言）
- 核心追问：**自然语言是可计算的吗？如何判断计算机是否"理解"了语言？**

### 2. 学科的产生与发展（关键时间线）

| 时间 | 事件 | 意义 |
|------|------|------|
| 1946 | 第一台计算机 **ENIAC** 诞生 | 技术基础 |
| 1947.3.4 | **Warren Weaver** 提出"用计算机翻译是否不可想象" | MT 思想萌芽 |
| 1949.7 | **维纳 N. Wiener**（控制论创始人，1894–1964）"Translation" 备忘录 | — |
| **1954** | **Georgetown–IBM** 用 IBM-701 实现首个 MT 系统（俄译英，250 词 + 6 条规则） | MT 第一次公开演示，引发热潮 |
| **1956** | **达特茅斯会议**（人工智能夏季研讨会） | **自然语言理解 (NLU)** 成为 AI 核心问题之一 |
| 1962 | **ACL**（国际计算语言学学会）成立 | 学科组织化 |
| 1965 | **ICCL**（国际计算语言学委员会）成立 | — |
| 1964 / 1966.11 | **ALPAC 报告**（"语言与机器"） | "计算语言学"术语首次进入官方学术报告；MT 热潮降温 |
| **1970–80s** | 计算机网络普及，语言工程兴起 | **"自然语言处理 (NLP)"** 术语诞生 |
| 70/80s | **中文信息处理 (CIP)** 出现 | 以汉语为处理对象 |

> 金句："自然语言处理是人工智能皇冠上的明珠。"

### 3. 核心术语辨析

| 术语 | 全称 | 提出年代 | 侧重 |
|------|------|----------|------|
| **NLU** | Natural Language Understanding | 1950s | 探索人类语言能力与认知过程本质；AI 早期研究领域 |
| **CL** | Computational Linguistics | 1960s | 用形式化计算模型分析/理解/生成语言；**偏基础理论** |
| **NLP** | Natural Language Processing | 1970–80s | 用计算机技术处理语言文本（词法/句法/语义/语用），**偏工程实现** |
| **HLT** | Human Language Technology | 1980s | 统称人类语言技术 |
| **CIP** | Chinese Information Processing | 70/80s | 中文（汉语）信息处理 |

> NLU、CL、NLP、HLT 三者研究对象交叉，界限随时代演变；**CL 偏理论、NLP 偏技术系统**。

### 4. 研究内容（多模态生态）

```
语音技术 + 文本处理 + 图像/视频 + 神经/认知科学
   │          │           │              │
 语音识别     分类/情感    OCR            语言学
 语音合成     检索/校对    文档分析        语言认知
 说话人识别   机器翻译     多模态交互      计算智能
```

- **理解与生成**两大主线任务：机器翻译、自动摘要、问答系统、自动写作……
- 分析处理：文本分类、情感分类、观点挖掘、主题发现、信息检索……

### 5. 应用需求

- 人类以语言文字记载的知识占知识总量 **80% 以上**。
- "任意时间、任意地点、任意语言"的无障碍自由通讯是人类梦想。
- HLT 广泛用于：国家与公共安全、国民经济发展、社会生活服务、国际交流合作。
- 例：已与 152 国、32 个国际组织签合作文件，涉及 110+ 种语言（截至 2023.6）。

## 二、问题与挑战

### 1. 歧义（ambiguity）—— 不确定性无处不在

- **词法歧义（分词）**：`门把手弄坏了` → 门/把/手…、门把手/…、门/把手/…
  - `自动化研究所取得的成就` → 自动化/研究所/…、自动化/研究/所/…
- **句法/结构歧义**：`I saw a boy in the park with a telescope.`
- **语义歧义**：`喜欢乡下的孩子`、`夏天/冬天能穿多少穿多少`
- **语用 / 隐喻 / 缩略语**：把权力装进制度的笼子、老虎苍蝇一起打、像石榴籽一样团结。

### 2. 大量未知语言现象

- **新词 / 未登录词**：给力、内卷、非典、新冠肺炎；人名地名（不来梅、Tony Blair=布莱尔）。
- **新含义**：苹果、奔腾、老虎、后台（词义漂移/引申）。
- **新用法 / 新句型**：被长工资、很中国、百度一下（非规范表达）。

### 3. 知识表示与获取的复杂性

- 知识（尤其常识）**模糊、隐蔽、密切关联**，且通过自然语言描述。

### 4. 跨语言概念差异

- 历史、文化、宗教、思维方式差异 → 语义概念不对等。

> 总命题：**如何从大量复杂多样的不确定性中寻找确定性结论？**

### 5. 关于"理解"的标准 —— 图灵测试（Turing Test）

- 图灵设计的"模仿游戏"：通过 **act（表现）/ react（反应）/ interact（相互作用）** 评估系统智能，与有意识的人比较。

### 6. 人脑如何学习、记忆、理解语言？

- 涉及语言学、心理学、认知科学、神经科学、计算机科学、统计学/信息论、背景知识与常识…… 多学科交叉、远未解决。

## 三、技术方法

### 1. 三大主义范式（核心！）

| 范式 | 代表技术 | 基本思路 | 系统构成 |
|------|----------|----------|----------|
| **理性主义**（符号主义） | 规则、句法（Chomsky 1957《句法结构》、Tesnière 1953 依存） | 由代表性语料归纳语言规律，建立**符号规则系统**：设计规则（N+N→NP）、标注词典、推导/归约/消歧 | **知识库 + 推理系统** |
| **经验主义**（统计学习） | n-Gram、HMM、SVM、ME、CRFs | 用大规模真实数据，统计语言规律（概率）；统计单元为**离散事件**（词、短语、词性） | **数据 + 标注 + 统计模型** |
| **连接主义**（神经网络） | CNN/RNN/LSTM/Attention/Transformer | 超大规模数据，用**连续实数向量**表示语言单元，精细刻画大范围上下文，统一模型框架 | **大数据 + 大模型 + 大算力** |

### 2. 以机器翻译为例：三种方法对比

#### ① 基于规则的方法（分析—转换—生成）
- 流程：分词词性标注 → 句法分析 → 结构转换（查双语词典）→ 译文生成。
  - 例：`桌子上有一本书。` → There is a book on the desk.
- ✅ 优点：保持原文结构、可追溯可溯源；规范句、已知现象效果好。
- ❌ 弱点：人工编规则、工作量大、主观性强、一致性差、难扩充、非规范语言差；移植性差、开发周期长。

#### ② 统计翻译方法（噪声信道模型 Noise Channel）
- 源句 S = s₁…sₘ，目标句 T = t₁…tₗ：
  $$T^* = \arg\max_T\, p(T)\cdot p(S|T)$$
  - $p(T)$ = **语言模型**；$p(S|T)$ = **翻译模型**。
- 需要双语平行句对训练（例：维汉平行语料）。
- ✅ 优点：不需深层分析，只要有足够多高质量双语句对即可，开发周期短。
- ❌ 弱点：高质量双语句对难收集；长复杂句译文差；语义一致性无保证；测试集与训练集差异大/出现生词时质量骤降。

#### ③ 神经网络方法（Transformer 端到端，统一框架）
- token 用连续实数向量表示；$\hat{e}_i = \arg\max\, p(e_i\mid e_1…e_{i-1}, C)$。
- **不同语言、不同模态、不同任务几乎无差别处理**（统一框架）。
- ✅ 优点：不需分析源句，仅凭大规模单语 + 双语词典/句对 + 提示指令即可；译文流畅，缩略语/术语处理好。
- ❌ 弱点：高质量数据/词典/指令难收集；**流畅但语义错误、"幻觉"/"无中生有"** 问题。

### 3. 大语言模型时代（2022.11 起）

- **2022.11.29 OpenAI 发布 ChatGPT (GPT-3.5)**：以自然语言提问即可获高质量回答 → 范式转变。
- 后续里程碑：GPT-4（2023.3）、Sora（2024.2）、GPT-4o（2024.5）、**o1/o3**（2024.9–2025.1）。
- 开源/他山：LLaMA（2023.3 → LLaMA4 2025.4）、Vicuna、StableLM、Grok 4（2025.7）。
- 国产：豆包、千问、文心一言、星火、ChatGLM、**DeepSeek**……
- 能力例证（o1）：AIME 83.3 / 96.7(o3)；GPQA Diamond 78%（超人类博士 69.7%）；IOI 超金牌门槛；Codeforces 89%。

### 4. 方法演进总览

```
Template-based → Rule-based(40年) → Statistical(20年) → Deep learning → 大模型
（理性主义）        （经验主义）         （连接主义）
"To be or not to be, this is a question."
```
- MT 关键节点：IBM-701(1954) → ALPAC(1966) → TAUM-METEO(1976) → EBMT(1984) → SMT(1989) → GIZA/GIZA++/Pharaoh/Moses → Google翻译(2006)/百度翻译(2011) → **NMT(2013)** → ChatGPT(2022.11)。

## 七、习题要点

1. 选取中英文句子，分析**词性兼类**与**一词多义**。
2. 对比**汉语与英语**句子结构差异。
3. 对比日常对话 / 网络聊天 / 网络新闻 / 规范书面文本（《人民日报》）的用词与句法差异。
4. 举例说明除大模型外，你日常用到的 NLP 系统/工具。
5. 测试 3 个机器翻译系统，对比不同语言对、不同系统译文差异与问题。
6. 测试 ≥5 个大模型，用第 5 题的句子测试翻译，与专业翻译系统对比分析。
