# 第08章 知识增强的大语言模型 —— 总结

> 解决 LLM 的**幻觉**等问题:用**检索增强生成(RAG)** 与**知识图谱**等外挂知识库为大模型"补脑"。

---

## 一、问题概述

### 1. 大模型的缺陷 ⭐
- **幻觉**(Hallucination)
- **信息过时**
- **参数化知识效率低**
- 缺乏专业领域**深度知识**
- **推理能力弱**

### 2. 实际应用需求
领域精准问答、数据频繁更新、生成内容**可解释可溯源**、成本可控、数据隐私保护。

### 3. 幻觉的类型 ⭐
| 类型 | 含义 |
|------|------|
| **语境冲突性幻觉(忠实性幻觉)** | 生成内容与之前生成内容/用户输入相冲突 |
| **与事实相冲突的幻觉** | 生成内容有悖客观事实 |
| **无中生有** | 完全捏造 |

### 4. 幻觉原因分析
- **缺乏知识或内化错误知识**:把相关性(位置接近/高度共现)误解为事实;偏向肯定测试样本、复制放大幻觉;训练数据含模仿性/重复性/偏见/过时。
- **高估自己能力**:大模型正确/错误答案分布熵相似,生成错误时同样"自信"。
- **对齐过程问题**:预训练缺先决知识时,错误对齐促发幻觉;**谄媚(sycophancy)**——偏向用户观点而非正确答案。
- **生成策略**:局部最优化(标记预测)≠全局最优(序列);top-p/top-k 抽样引入随机性。

### 5. 幻觉评估
- 事实性陈述的判别标准?如何度量幻觉指标?(需数据集)

## 二、检索增强生成(RAG)⭐⭐

### 1. 概念
- LLM 回答/生成时,**先从大量文档检索相关信息,再基于检索结果生成**。
- **不必为每个任务重新训练整个大模型**,只需外挂知识库;尤其适合**知识密集型**任务。

### 2. 大模型优化三方式(外挂知识库 vs 知识参数化)
| 方式 | 场景 |
|------|------|
| **提示工程 Prompt Engineering** | — |
| **检索增强 RAG** | 用户需要特定信息 |
| **指令微调 Instruct/Fine-tuning** | 模型需要掌握一种能力 |

### 3. RAG vs Fine-tuning 对比 ⭐
| 因素 | RAG | Fine-tuning |
|------|-----|-------------|
| 外部知识利用 | 擅长,适合结构化/非结构化库 | 需构造监督数据内化,不适用频繁更改 |
| 知识更新 | 直接更新检索库,适合动态环境 | 重新微调,耗资源 |
| 训练数据要求 | 对数据加工要求低 | 依赖高质量数据集 |
| 可解释性/可溯源 | 高,可追溯特定数据源 | 低,黑匣子 |
| 可扩展性 | 高,动态衔接多数据源 | 低,新知识需重新微调 |
| 耗时 | 多次检索有延迟 | 微调后无需检索,响应快 |

### 4. RAG 基本流程(三步)⭐⭐
**① 构建数据索引**:
- 文档切成均匀**块**(chunk);
- 用编码模型为每块生成 **Embedding**;
- 存入**向量数据库**。
  > Small-to-Big:句子级嵌入,生成时扩大窗口。

**② 检索**:用向量相似度检索与问题**最相关的 k 个文档**。

**③ 生成**:原始 Query + 检索文本组合 → 输入 LLM → 回答。

### 5. 三个问题(检索什么/何时/如何用)
- **检索什么**:词元/词组/句子/段落/实体/知识图谱。
- **何时检索**:单次/每 token/每 n token(phrase)/自适应检索。
- **何时增强**:预训练/指令微调/推理。
- **如何用**:输入数据层/模型中间层/输出预测层。

### 6. 使用场景
数据长尾、知识更新频繁、回答需验证追溯、领域专业、隐私保护。代表方法:RETRO、REALM、ATLAS、RAG(Lewis 2020)、kNN-MT、FLARE、kNN-Prompt 等(问答/事实验证/对话/翻译/代码/文摘/推理/情感)。

## 三、其他改进方法:动态知识引导

### 1. 知识表示方法
逻辑表示(命题/谓词逻辑)、产生式(IF…THEN)、框架、面向对象、**语义网 → 知识图谱**、本体表示。

### 2. 知识图谱(回顾第4章)
- **构建**:实体识别(边界/类别)、**实体消歧**(迈克·乔丹=篮球/教授?)、关系定义与抽取(关系分类)、事件抽取(限定域/开放域/事件关系)。
- **开源知识图谱**:英文 WikiData、YAGO、DBpedia;中文 OpenKG(openkg.cn)。
- **知识推理**:**归纳推理**、**演绎推理**((姚明,出生地,上海)+(上海,属于,中国)⇒(姚明,是中国人))。

### 3. 动态知识引导的自动写作
- **问题**:LLM 生成文本过于关注提示中的实体,信息丰富度低(信息丰富度排名:人工 1.60 > GPT 各版本 > LLAMA-13B 4.00)。
- **改进思路**:在知识图谱中找**子图约束文本生成**(一跳关系的子图),增强信息丰富度、避免偏离主题。

### 4. 评价指标
- 文本质量:**PPL、Dist-3、ROUGE-L、BERTScore**。
- 信息丰富度:**知识约束实体数量(KC-E)**——生成文本中与输入提示实体在知识图谱中存在直接关系(至少一条边)的实体数。相关性最好(斯皮尔曼 0.554)。

## 本章小结(背诵要点)

- **缺陷**:幻觉、信息过时、参数化知识效率低、缺专业深度、推理弱。
- **幻觉三类**:语境冲突(忠实性)、与事实冲突、无中生有;原因:缺知识/高估能力/对齐问题(谄媚)/生成策略。
- **RAG**:先检索后生成;三步(分块+Embedding 入向量库 → 向量相似检索 k 个 → Query+检索文本输入 LLM);适合知识密集/动态更新;**可溯源可解释**;外挂知识库无需重训。
- **RAG vs 微调**:RAG 擅长外部知识/动态/可解释/可扩展但有延迟;微调内化能力但耗资源、黑匣子。
- **知识图谱增强**:子图约束生成提升信息丰富度;指标 PPL/Dist-3/ROUGE-L/BERTScore + **KC-E**(知识约束实体数)。
