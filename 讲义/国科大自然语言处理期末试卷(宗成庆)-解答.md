## 一、完成下列各题（20 分）

### (1) 给出一个汉语句子，说明连词引起歧义结构

**例句：「我和他的老师」**

连词「和」的作用范围（管辖域）不同导致两种句法结构：

- **切分一**：`[我] 和 [他的老师]`
  - 「和」连接「我」与「他的老师」两个并列名词短语；
  - 语义：一个人（我）与另一个人（他的老师）。
- **切分二**：`[我和他] 的 老师`
  - 「和」连接「我」与「他」，整体作定语修饰「老师」；
  - 语义：我和他共同的老师。

> 类似例：「中国和俄罗斯的关系」→ `(中国) 和 (俄罗斯的关系)` vs `(中国和俄罗斯) 的关系`。
> **考点**：歧义（结构歧义）是 NLP 的核心困难之一；连词「和/与/或/同」的辖域歧义是典型。

### (2) 字符串 `aomnad` 与单词 `command` 的编辑距离

编辑距离（Levenshtein Distance）= 把串 X 变成 Y 所需**插入、删除、替换**的最少操作数。动态规划：

```
        ε   c   o   m   m   a   n   d
    ε   0   1   2   3   4   5   6   7
    a   1   1   2   3   4   4   5   6
    o   2   2   1   2   3   4   5   6
    m   3   3   2   1   2   3   4   5
    n   4   4   3   2   2   3   3   4
    a   5   5   4   3   3   2   3   4
    d   6   6   5   4   4   3   3   3
```

**编辑距离 = D[6][7] = 3**

一种最优对齐（3 步）：

```
a o m n a d          操作
c o m m a n d        ─────
│ │ │ │ │ │          a → c   (替换)
a→c                  
  o=o  (匹配)
    m=m  (匹配)
      n→m  (替换)     n 替换为 command 的第 2 个 m
        a=a  (匹配)
          ＋n (插入)  插入 n
            d=d  (匹配)
```

> **结论**：`ed(aomnad, command) = 3`。
> **考点**：噪声信道模型在拼写检查中的应用；编辑距离的定义与 DP 求解；阈值 t 的作用（限定截取范围与编辑距离上界）。

### (3) 由有限状态自动机 M 构造正则文法 G，使 L(G) = T(M)

> ⚠️ 原图自动机的状态/转移辨识不清，下面给出**通用构造方法**，套用到具体 M 即可。

**构造方法（FA → 右线性正则文法）**

设 M = (Σ, Q, δ, q₀, F)，令文法 G = (N, Σ, P, S)，其中 N = Q（状态作非终结符），S = q₀：

1. 对每个转移 `δ(A, a) = B`，加产生式 `A → aB`；
2. 若 `δ(A, a) = B` 且 `B ∈ F`（B 为终态），再加 `A → a`（直接生成终极符）；
3. 若 `q₀ ∈ F`，加 `S → ε`。

**示例**：M 接受所有以 1 结尾的 0/1 串。Q = {q₀, q₁}，Σ = {0,1}，q₀ 起始，F = {q₁}，
δ(q₀,0)=q₀，δ(q₀,1)=q₁，δ(q₁,0)=q₁，δ(q₁,1)=q₁。

构造得：

```
q₀ → 0 q₀ | 1 q₁ | 1
q₁ → 0 q₁ | 1 q₁ | 0 | 1
```

> **考点**：正则文法（3 型）与有限状态自动机的等价性 T(M) = L(G)；右线性规则形式 `A → xB` 或 `A → x`。

### (4) 复杂特征集 A ∪ B = ？

```
A = [ POS = NN          B = [ Lex   = boy
      Lex  = boy               Num   = Single
      Sem  = Agent ]           Case  = Subject ]
```

**合一（Unification）运算**：对同名属性要求值一致（冲突则失败），不同名属性合并。

- 共有属性 `Lex`：A、B 均为 `boy`，一致 ✓；
- 其余属性互补，直接合并。

```
A ∪ B = [ POS  = NN
          Lex  = boy
          Sem  = Agent
          Num  = Single
          Case = Subject ]
```

> **考点**：复杂特征集（feature structure）与合一运算，用于基于合一的句法分析；合一满足交换律、结合律，冲突则不可合一。

### (5) 写出句子 "I study English every day." 的所有 3 元文法（Tri-grams）

为统计句首/句尾概率，引入边界符 `<s>`、`</s>`（trigram 通常再加一个起始符）。

句子词序列（加边界）：`<s> <s> I study English every day </s>`

所有 3 元文法（共 **6** 个）：

| # | Trigram |
|---|---------|
| 1 | `<s> <s> I` |
| 2 | `<s> I study` |
| 3 | `I study English` |
| 4 | `study English every` |

> **考点**：n 元文法模型；trigram 条件概率 `P(wᵢ | wᵢ₋₂ wᵢ₋₁)`；边界符的作用。

## 二、阐述题（10 分）：设计一种新的、可计算的机器翻译译文质量自动评价方法

### 方法：语义加权 BLEU（Semantic-weighted BLEU, SW-BLEU）

**动机**：传统 BLEU 只做表面 n-gram 精确匹配，对同义替换（如 *important/crucial*）、语态转换、词序调整等合法译法过度惩罚；而人工评价又不可大规模复用。需要一个**仍可自动计算**但能感知**语义**的方法。

**步骤与公式**

1. 对参考译文 R、机器译文 H 分别分词，得词序列。
2. 取 H 中每个 n-gram，在 R 中查找匹配，按匹配类型加权：
   - **精确匹配**（n-gram 完全相同）：权重 `w = 1`；
   - **语义近似匹配**（逐词词向量余弦相似度均值 ≥ 阈值 τ）：权重 `w = λ`（0 < λ < 1，如 0.6）；
   - 无匹配：`w = 0`。
3. 加权 n-gram 精度：
   $$P_n^{sem} = \frac{\sum_{g\in H_n} w(g)}{\text{H 中 n-gram 总数}}$$
4. 几何平均 + 长度惩罚（沿用 BLEU）：
   $$\text{SW-BLEU} = BP \cdot \exp\!\left(\sum_{n=1}^{N} \alpha_n \log P_n^{sem}\right),\quad BP = \min(1,\, e^{1-|R|/|H|})$$

（可选）再与目标语言模型困惑度（流畅度项）线性组合，兼顾通顺度。

**合理性论证**

- **可计算**：n-gram 计数、词向量（预训练）、余弦相似度、BP 均为确定可程序化的运算，无需人工介入；
- **语义感知**：词向量把“同义不同形”的译法纳入奖励，缓解 BLEU 的表面惩罚，更贴近人工“忠实度”判断；
- **可调**：阈值 τ、权重 λ、阶数 N、组合系数 αₙ 均可据语料调参，便于优化；
- **兼容性**：当 λ = 0（关闭语义匹配）时退化为标准 BLEU，便于对照。

> **考点**：译文评估的三大要素（流畅度、充分性、语义保持）；客观评测方法（WER、BLEU、METEOR、基于深度学习的方法）；新方法须“可计算”的设计要求。

## 三、综合题（15 分）

### 1. 分词与词性标注系统的准确率、召回率、F1

**原句**：`2013年轻工业产品质量大幅度提升。`

**标准答案（Gold，7 词）** vs **系统输出（Sys，9 词）**，按字符跨度对齐：

| Gold 词 | 跨度 | Sys 词 | 跨度 | 匹配 |
|---------|------|--------|------|------|
| 2013年 | [0,4] | 2013 | [0,3] | ✗ |
| 轻工业 | [5,7] | 年轻 | [4,5] | ✗ |
| 产品 | [8,9] | 工业 | [6,7] | ✗ |
| 质量 | [10,11] | 产品 | [8,9] | ✓ |
| 大幅度 | [12,14] | 质量 | [10,11] | ✓ |
| 提升 | [15,16] | 大 | [12] | ✗ |
| 。 | [17] | 幅度 | [13,14] | ✗ |
| | | 提升 | [15,16] | ✓ |
| | | 。 | [17] | ✓ |

正确匹配词数 **C = 4**（产品、质量、提升、。）。Gold 词数 G = 7，Sys 词数 S = 9。

$$P = \frac{C}{S} = \frac{4}{9} \approx 44.4\%,\quad R = \frac{C}{G} = \frac{4}{7} \approx 57.1\%$$

$$F_1 = \frac{2PR}{P+R} = \frac{2C}{G+S} = \frac{8}{16} = 0.5\;(50\%)$$

> **结论**：准确率 ≈ 44.4%，召回率 ≈ 57.1%，F1 = 0.5。
> **考点**：分词评测以**词的字符跨度**完全一致计为正确；P/R/F1 公式与 F1 = 2C/(G+S) 捷径。

## 四、句法分析（15 分）

**文法 G(S)**：
① S → P VP　② VP → PP V　③ VP → Prop VP　④ PP → Prop
⑤ VP → VP V　⑥ VP → V Aux　⑦ VP → VP N　（⑧ PP → Prop）

**词典**（兼类用逗号）：
`从:Prop  从小:Prop  他:P  会:V,N  学:V  学会:V,N  了:Aux  解:V,N  了解:V  解题:V  小:A  小学:N  题:N`

**给定句子**：`他从小学会了解题`（不计句号）

### (1) 正向最大匹配 vs 逆向最大匹配

词典最长词长 = 2（学会/了解/解题/小学/从小），取最大词长 = 2。

**正向最大匹配（FMM）**：从左向右每步取最长匹配

| 位置 | 候选 | 结果 |
|------|------|------|
| 1 | 他从?否 → 他 | **他** |
| 2 | 从小 ✓ | **从小** |
| 4 | 学会 ✓ | **学会** |
| 6 | 了解 ✓ | **了解** |
| 8 | 题 | **题** |

**FMM = `他 / 从小 / 学会 / 了解 / 题`**

**逆向最大匹配（BMM）**：从右向左每步取最长匹配

| 位置 | 候选 | 结果 |
|------|------|------|
| 8 | 解题 ✓ | **解题** |
| 6 | 会了?否 → 了 | **了** |
| 4 | 学会 ✓ | **学会** |
| 2 | 从小 ✓ | **从小** |
| 1 | 他 | **他** |

**BMM = `他 / 从小 / 学会 / 了 / 解题`**

> 两者在「了解题」处分歧：FMM → `了解/题`，BMM → `了/解题`。这是经典的分词歧义。
> **考点**：机械分词（最大匹配法）的正向/逆向实现、歧义与局限。
