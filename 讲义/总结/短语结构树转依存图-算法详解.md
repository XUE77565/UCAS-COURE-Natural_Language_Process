# 短语结构树 → 依存关系图：算法详解

> 本文是 [第04章 依存关系分析-详解.md](第04章%20依存关系分析-详解.md) 第 3 节的**深化版**：把“怎么转”讲成一套可执行算法——**中心词发现（Head Finding）+ 中心词渗透（Head Percolation）**，并处理并列、的字结构、助动词等难点。
>
> 适合回答「详细讲解短语结构树如何转化成依存图」这类问题。

---

## 0. 两种表示的本质差异

| | 短语结构树（Constituency / Phrase Structure）| 依存关系图（Dependency）|
|---|---|---|
| 节点 | 短语类型（NP/VP/PP/S）+ 词 | **只有词** |
| 边 | 父→子的「直接组成」| 支配词→从属词的「依存」|
| 讲什么 | **包含**（NP 里有什么）| **谁管谁**（哪个词是中心）|
| 形态 | 多层嵌套 | 扁平、词汇化 |
| 例子 | `S→NP VP` | `读 → 他` |

**桥梁**：每个短语都有一个**中心词（head）**。短语在依存图里就用它的中心词“代表”。整棵短语树转依存图，本质就是**沿中心词自底向上做一次归约**。

> 这套“给每个非终结符标注中心词”的做法，在统计句法分析里叫 **Head Lexicalization（Collins 1997/1999）**——短语结构分析器要利用中心词信息时，做的就是这件事。

---

## 1. 核心思想：中心词渗透（Head Percolation）

两条递归定义：

1. **叶子（词）的中心词 = 它自己**。
2. **非终结符 $X$ 的中心词 = $X$ 的「中心子节点」$c^*$ 的中心词**（中心子节点由「中心词规则」决定，见 §3）。

转换规则（建弧）：

> 对 $X$ 的每个**非中心子节点** $c$，建一条依存弧：
> $$\text{head}(X) \longrightarrow \text{head}(c)$$
> 弧上关系标签由（$c$ 的类型、$X$ 的类型、位置）决定（见 §4）。

最终：整棵 $S$ 的中心词 = **ROOT**（句子的主动词）。

一句话口诀：**自底向上找中心，中心支配其余子成分的中心；主动词登顶为根。**

---

## 2. 形式算法（伪代码）

```
Convert(X):                      # 输入短语树节点 X，返回 X 的中心词
    if X 是词(叶子):
        return X                 # 叶子的中心=自己

    c* = FindHead(X)             # 用中心词规则找出中心子节点
    H  = Convert(c*)             # 中心词渗透：H = 中心子节点的中心词

    for 每个 X 的子节点 c ≠ c*:
        h_c = Convert(c)         # 递归转换子树
        新建依存弧  H ──label──→ h_c
        label = RelLabel(c, X, 位置)   # §4 的关系映射

    return H

# 主调用
ROOT = Convert(S_root)
```

**复杂度**：$O(n)$，每个节点访问一次。
**确定性**：给定中心词规则表，转换结果唯一。

---

## 3. 中心词规则表（Head Rules）

中心子节点的选取**取决于短语类型**，且与语言有关。下面给汉语（CTB 风格）和英语（Collins 风格）常用规则。

### 3.1 汉语中心词规则

| 短语类型 | 中心子节点 | 方向/说明 |
|---|---|---|
| **S / IP**（句）| 谓语动词（递归到 VP 的中心）| 整句根 = 主动词 |
| **VP** | 动词（VV/VE/VA）| 助动词在前时仍取**主要动词** |
| **NP** | 最右的名词（NN/NR/NT）| “XP 的 N”中 **N** 是中心 |
| **PP** | **介词 P**（在/把/被/对…）| ⭐ 介词才是中心，不是后面的名词 |
| **ADJP** | 形容词 JJ | |
| **ADVP** | 副词 AD | |
| **QP / CLP** | 数词 / 量词（M）| “三本”→量词“本”是中心 |
| **DP** | 限定词（DT：这/那）| |
| **CP**（含“的”/标句词）| **助词“的”/标句词**（DEG/CS）| “红色的苹果”里 CP 的中心是“的” |
| **LCP** | 方位词 LC（上/下/里）| |
| **FRAG**（片段）| 取唯一实词 | |

### 3.2 英语中心词规则（Collins'99 节选）

| 短语 | 中心查找 |
|---|---|
| VP | 从右向左找 `MD / VBD / VBZ / VBP / VB / VBG / VBN` |
| NP | 从右向左找 `NN / NNS / NNP / NNPS`，否则取最右子节点 |
| PP | 取介词 `IN`（from/right）|
| S / SBAR | 取主要动词（递归）|
| ADJP | 取最右形容词 |

> 记忆要点：**英语 NP/V/ADJP 多从右找**（中心词在右），**汉语 NP 也多在右**；**PP 的中心永远在介词上**。

## 4. 关系标签怎么定（RelLabel）

弧的标签由**子成分的语法功能**决定，可由 `(子类型, 父类型, 位置)` 映射：

| 子成分 $c$ 在父 $X$ 中的角色 | 依存关系 | 标签（LTP/UD）|
|---|---|---|
| NP 在 S/IP 中、主动词之**左** | 主语 | SBV / nsubj |
| NP 在 VP 中、动词之**右** | 宾语 | VOB / obj |
| PP 修饰 VP | 状语（处所/工具/时间）| ADV / obl |
| PP / CP 修饰 NP | 定语 | ATT / nmod |
| ADJP / ADVP 修饰 V | 状语 | ADV / advmod |
| ADJP / CP 修饰 N | 定语 | ATT / amod |
| 助词“的” | 的字结构 | DE / mark |
| 连词“和、与” | 并列连词 | COO/LAD / cc |
| 并列的后项 | 并列 | COO / conj |
| 助动词/能愿动词 | 辅助/状语 | ADV(AUX) / aux |
| 标点 | 标点 | WP / punct |

> **箭头方向**：中心词（被修饰者）→ 修饰语（从属者）。例：动宾是 `动词→宾语`，定中是 `中心名词→定语`。

## 5. 完整例子（含并列结构）

**句子：他吃苹果和香蕉。**

短语结构树：

```
                 S
          ┌──────┴──────┐
         NP            VP
         他     ┌───────┼───────┐
                VV      NP      PU
                吃    ┌──┴──┐    。
                      NN  CC  NN
                     苹果  和  香蕉
```

### 第 1 步：自底向上找中心词

- `吃`(VV)→**吃**；`苹果`(NN)→**苹果**；`和`(CC)→**和**；`香蕉`(NN)→**香蕉**
- `NP(苹果和香蕉)`：并列结构，取**第一个并列项** `苹果` 为中心
- `VP(吃 苹果和香蕉 。)`：中心是动词 **吃**
- `S`：中心 = VP 的中心 = **吃 = ROOT**

### 第 2 步：建弧（中心支配非中心子节点的中心）

- `NP(他)` 是 S 的非中心子节点 → `吃 → 他`（SBV 主语）
- `NP(苹果和香蕉)` 是 VP 的非中心子节点，其中心是苹果 → `吃 → 苹果`（VOB 宾语）
- `PU(。)` → `吃 → 。`（WP 标点）
- 在 `NP(苹果和香蕉)` 内部，中心=苹果，非中心子节点是 `和`、`香蕉`：
  - `苹果 → 和`（LAD 并列连词）
  - `苹果 → 香蕉`（COO 并列）

### 第 3 步：成图

```
                    吃  (ROOT)
                 /   /   \    \
               他  苹果   。    
                   /  \
                  和  香蕉
```

| 弧（支配→从属）| 关系 |
|---|---|
| 吃 → 他 | SBV 主语 |
| 吃 → 苹果 | VOB 宾语 |
| 吃 → 。 | WP 标点 |
| 苹果 → 和 | LAD 并列连词 |
| 苹果 → 香蕉 | COO 并列 |

**自检**：单根 `吃` ✓；每词单一父结点 ✓；弧不交叉（可投射）✓。

## 6. 难点专题

### 难点 1：介词短语——head 是介词不是名词 ⭐

“在阳台读书”：`PP(在 阳台)` 的中心是**介词“在”**，不是“阳台”。

```
读 → 在 (ADV) ;  在 → 阳台 (POB)     ✓ 正确
读 → 阳台                            ✗ 跳过介词，错
```

> 这是最高频的错误，见 [第04章 依存详解 §3 易错点](第04章%20依存关系分析-详解.md)。

### 难点 2：“的”字结构——把“的”当独立词处理

**句子：红色的苹果很甜。** 词序列：`红色 / 的 / 苹果 / 很 / 甜`。

短语树片段：`NP → CP(红色 的) + NN(苹果)`，`CP → JJ(红色) + DEG(的)`。

转换（**token 级，逐词建弧**）：

- `NP` 中心 = 最右名词 **苹果**
- `CP` 中心 = 助词 **的**
- 苹果（NP 中心）支配非中心子节点 CP 的中心“的” → `苹果 → 的`（DE）
- 在 CP 内，“的”支配非中心子节点“红色” → `的 → 红色`（ATT）
- `S` 中心 = 谓语 **甜**（无动词时谓语形容词当根）
- `苹果 → 甜`（SBV）、`很 → 甜`（ADV）

```
              甜 (ROOT)
             /    \
           苹果    很
            |
           的      ← DE（的字结构）
           |
          红色     ← ATT（定语）
```

> 简化视角下也可把“红色的”当一个定语整体挂“苹果”（ATT），但在**短语树转换**里是 token 级的，必须把“的”单独建弧。两种画法考试都接受，**关键别把“红色”直接挂“甜”**。

### 难点 3：助动词 / 能愿动词——主要动词仍是中心

“他**会**去” / “**已经**吃了”：

- 能愿动词“会”、副词“已经”、助词“了”都**从属于**主要动词，主动词（去/吃）仍是 ROOT。
- `去 → 会`（ADV/AUX）、`吃 → 已经`（ADV）、`吃 → 了`（CMP 动补/助词）。

### 难点 4：兼类与中心词消歧

“从小”可作副词或介词；“学会”可作动词或名词。中心词规则要靠**词性标注**消歧；词性错 → 中心词找错 → 整个子树依存关系错。这是为什么分词+词性标注是句法分析的前置步骤。

### 难点 5：长距离依存与非投射

“我昨天**买的**那本书**很有趣**”——定语从句跨多个词修饰“书”，转换后依存弧可能跨越其他词，产生**非投射（non-projective）**结构。移进-归约（Arc-eager 等）决策式算法处理不了非投射，需图算法（如 Chu-Liu-Edmonds 最大生成树）。

### 难点 6：空节点 / trace（英语为主）

“The book$_i$ I read $t_i$”——宾语移位后留下 trace。短语树里有空节点 `-NONE-`，转换时要**把 trace 与其先行词合并**，恢复正确的依存（read → book），否则依存断裂。

---

## 7. 转换的性质（简答可能问）

1. **确定性**：给定中心词规则表，一对一映射，结果唯一。
2. **保持可投射性**：若短语结构树的所有成分**不交叉嵌套**（正常 CFG 推导都满足），则转换出的依存树**可投射**；长距离依存/移位才会非投射。
3. **信息有损**：
   - 依存树**丢失**了短语的边界与类型（你不再知道“苹果和香蕉”是一个 NP）；
   - 但**新增**了 head 关系（哪个词是中心）。
   - 故短语结构树 → 依存树**不可逆**（多棵短语树可对应同一依存树）。
4. **与 Robinson 四公理等价的自检**：单根 + 单一父结点 + 连通无环 + 可投射 = 合法依存树。

---

## 8. 一页小结（考场速查）

```
转换 = 中心词渗透 + 建弧

  Convert(X):
    1. 叶子 → 返回自己
    2. c* = FindHead(X)      # 中心词规则表
    3. H   = Convert(c*)     # 中心词
    4. 对每个非中心子 c：
         建 H → Convert(c)，标关系
    5. 返回 H
  ROOT = Convert(S)
```

**四步作答法**：
1. 标每个短语的中心词（自底向上）；
2. 中心支配非中心子节点的中心 → 建弧；
3. 标关系（主谓/动宾/状中/介宾/定中/并列/的字）；
4. 四公理自检（单根、单父、连通、可投射）。

**三大易错**：① PP 的 head 是介词；② “的”要单独建弧；③ 主动词（非助动词/能愿动词）才是 ROOT。
